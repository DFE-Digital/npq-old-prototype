The wizard paths and wizard helpers files make it easy to build and iterate a page-per-thing journey.

The paths file defines:

  • the order of pages
  • fork points in the journey, where the journey must diverge based on a given answer

You can rearrange the list of pages to quickly change the order of pages in a journey.

The templates automatically generate:

  • a back link to the previous page
  • a ‘Continue’ button which submits the form
  • a form action that submits to the current page
